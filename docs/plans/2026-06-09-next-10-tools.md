# Next 10 Tools — the no-fluff plan (2026-06-09)

**PLAN COMPLETE -- 10/10 shipped (v1.12.0: 6, v1.13.0: 4).**

Ten tools, each justified by one of two hard signals — **mined demand** (independently needed by multiple games in the 51-game corpus) or **documented pain** (a session-killer in `BRIDGE_GOTCHAS.md` / `TROUBLESHOOTING.md`). ## Tier A — verified-unblocked scaffolds (APIs confirmed live this session)

### 1. `create_interactable`
- **What:** Generate a `Component, Component.IPressable` stub — `Press(Event)→bool` host-validated action, `Look`/`Hover`/`Blur` prompt hooks, `CanPress` gate, `GetTooltip` text, `IsProxy` guard. Optional `targetId` attach like the other scaffolds.
- **Why real:** 2x mined demand + interaction is the shared verb in nearly every genre recipe (shop-vendor table alone: 8 games, most via an `Interactable`/`IPressable` shape). It's the missing primitive between "scene exists" and "player can do something."
- **Verified:** `IPressable` full surface confirmed via `describe_type` (8 members, nested in `Component`).
- **Risk: easy.** Same `ScaffoldHelpers` pattern, ~120 lines of generator.

> **STATUS (2026-06-09): BUILT + VERIFIED** -- shipped in v1.12.0, compile-verified live.

> **STATUS (2026-06-09): BUILT + VERIFIED** -- shipped in v1.12.0, compile-verified live.

## Tier B — pain-driven workflow tools (fix documented session-killers)

### 5. `copy_asset_with_dependencies`
- **What:** Copy a model/material into the project **with its full dependency chain** (`.vmdl` + every `.vmat` + `.vtex` it references, recursively — or the compiled `_c` chain), into a **project-namespaced path**, refusing any destination that shadows a core tree (`models/citizen/**`, `materials/dev/**`).
- **Why real:** Kills gotchas **#4 (ERROR mesh)** and **#5 (shadow → endless recompile loop)** in one tool — the two asset-pipeline failures that each cost whole sessions and currently end with "restart and clean up by hand." This is the highest-leverage non-scaffold on the list.
- **Verify-gate:** `describe_type` on `Asset` / `AssetSystem` for the dependency-enumeration API (`GetDependencies`-shaped; in the sbox.game/api schema — **not yet live-verified, editor was down**). Fallback if no API: parse compiled-asset reference strings (the `_c` formats embed child paths).
- **Risk: medium.** File ops are already hardened (`TryResolveProjectPath`).

> **STATUS (2026-06-09): BUILT + VERIFIED** -- shipped in v1.12.0, compile-verified live.

### 6. `razor_lint`
- **What:** Static scan of project `.razor`/`.razor.scss` files for every known transpiler/UI footgun: switch-*expressions* in `@code`, emoji/non-ASCII in `@code`, root type-selector SCSS rules, `PanelComponent` in scene without a `ScreenPanel`/`WorldPanel` host (cross-checks the scene like `scene_validate` does), `BuildHash` missing on panels that render synced values.
- **Why real:** Gotcha #6 is four separate "valid code, opaque crash/silent no-op" traps — the worst debugging class there is, because `get_compile_errors` shows nothing useful. Direct sibling of the already-shipped `networking_lint`/`scene_validate` family, which proved the lint pattern works.
- **Risk: easy.** Mostly MCP-server-side text scan + one scene query.

> **STATUS (2026-06-09): BUILT + VERIFIED** -- shipped in v1.12.0, compile-verified live.

### 7. `sandbox_lint`
- **What:** Static scan of project `.cs` files for whitelist/sandbox violations *before* hotload: `System.Math`/`MathF`, `Array.Clone()`, `System.Net.*`, other known-blocked BCL members (seed list from gotcha #7 + CLAUDE.md; extensible table). Reports file+line, suggests the whitelisted swap (`MathX`, `.ToArray()`, …).
- **Why real:** Gotcha #7: whitelist rejections surface with **no file path**, masked by the broken-reference cascade — currently diagnosed by grepping the log after the compile already failed. This catches them before the compile, with a line number. The scaffold generators themselves shipped a `MathX` bug once (TOOL_BACKLOG note) — this lints our own output too.
- **Risk: easy.** Pure text scan; zero editor API.

> **STATUS (2026-06-09): BUILT + VERIFIED** -- shipped in v1.12.0, compile-verified live.

## Build protocol (same as the director trio — non-negotiable)
1. `describe_type` every API the generator will emit against (the NPC-brain generator shipped 2 bugs that passed eyeball inspection; live reflection is the only gate that caught them).
2. Generate → `trigger_hotload` → `get_compile_errors` → confirm the class is in the TypeLibrary.
3. `sandbox_lint` (tool #7) the generated output once it exists — the lint tools come early partly so they gate the later scaffolds.
4. One commit per tool; CHANGELOG + handler-count bump per release, not per tool.
